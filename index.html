State describes the internal state of the component. It is similar to props except that the state is defined within the component and is only accessible from within the component.

React components lifecycle

1.Initialization. This stage requires the developer to define properties and the initial state of the component.

2.Mounting. Mounting is the phase in which our React component mounts on the DOM

3.Updation. In this phase, the data of the component (state & props) updates in response to user events like clicking, typing and so on. This results in the re-rendering of the component.

4.Unmounting. This is the last phase in the component lifecycle. In this phase component is detached from the DOM container.

React routing

Router defines a set of routes and when a request comes to the application, Router matches the request with the routes. If some route matches the request URL, then this route is chosen to process the request.
Switch is responsible for grouping elements of the Route. It goes through the child elements and selects the first match in the URL path.

Each route represents a Route object. It has a number of attributes.

  1. path: the address pattern against which the requested URL will be matched
  2. element - the component that is responsible for processing the request for this route
Link is a special component that allows to follow links without reloading the page. It updates the URL defined by the "to" attribute. Router loads the desired component - all within a single page.
